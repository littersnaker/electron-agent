# Electron 客户端改造方案

## 总体思路

项目已经配置了 `output: "standalone"`，这意味着 Next.js 构建后会生成自包含的 `server.js`。Electron 主进程将启动这个 Next.js 服务器作为子进程，然后在 BrowserWindow 中加载页面。API 路由、LangGraph Agent 等服务端功能全部在 Electron 内部运行，无需外部服务器。

## 实施步骤

### 步骤 1：安装 Electron 相关依赖

安装以下依赖：
- `electron` - Electron 核心
- `@electron-forge/cli` + `@electron-forge/maker-squirrel` + `@electron-forge/maker-zip` + `@electron-forge/maker-deb` + `@electron-forge/maker-rpm` - Forge 打包工具链
- `electron-squirrel-startup` - Windows 自启动支持
- `electron-is-dev` - 区分开发/生产环境
- `electron-updater` - 自动更新

### 步骤 2：创建 Electron 主进程文件

创建 `electron/main.ts`：
- 启动 Next.js standalone 服务器作为子进程（开发模式用 `next dev`，生产模式用 `.next/standalone/server.js`）
- 创建 BrowserWindow，加载 `http://localhost:3000`
- 配置应用窗口（标题 "AI Chat"、窗口图标、默认尺寸）
- 集成 `electron-updater` 实现自动更新（检查更新间隔 1 小时，发现新版本自动下载并提示重启安装）

### 步骤 3：创建预加载脚本

创建 `electron/preload.ts`：
- 使用 `contextBridge` 暴露必要的安全 API
- 暴露 `app.getVersion()` 等基础信息
- 暴露 `platform` 信息供客户端使用

### 步骤 4：配置 Electron Forge

创建 `forge.config.ts`：
- 配置入口点为 `electron/main.js`（编译后的）
- 配置 makers：Squirrel (Windows NSIS)、ZIP (Windows portable)、DMG (macOS)、AppImage (Linux)
- 配置 packagerOptions：设置应用名称、图标路径等
- 配置 electron-updater 的发布目录

### 步骤 5：修改 package.json

- 添加 `"main": ".electron/main.js"` 入口
- 添加 Forge 脚本：`"electron:dev"`, `"electron:build"`, `"electron:make"`
- 添加构建相关配置

### 步骤 6：修改 next.config.ts

- 在 Electron 生产构建时禁用 `automaticVercelMonitors`（因为不是 Vercel 环境）
- 保持 `output: "standalone"` 不变

### 步骤 7：处理 API 密钥

由于选择"打包进客户端"，密钥将通过 `.env` 文件在构建时处理：
- 在 Electron 主进程中通过环境变量注入 `DASHSCOPE_API_KEY`、`GEMINI_API_KEY` 等
- 确保密钥不会被暴露到渲染进程

### 步骤 8：更新 .gitignore

添加 Electron 相关的忽略规则：
- `.electron/` - 编译输出
- `out/` - Forge make 输出

### 步骤 9：添加构建脚本

创建 `scripts/build-electron.ts` 用于：
1. 编译 TypeScript (`electron/main.ts` → `.electron/main.js`)
2. 运行 `next build`
3. 将 standalone 输出 + static 资源复制到 Electron 可访问的目录
4. 运行 Forge 打包

### 步骤 10：添加窗口图标

在 `public/` 目录下添加应用图标文件（如 `icon.png`），用于窗口标题栏和打包

## 文件变更清单

| 操作 | 文件 |
|------|------|
| 新建 | `electron/main.ts` - Electron 主进程 |
| 新建 | `electron/preload.ts` - 预加载脚本 |
| 新建 | `forge.config.ts` - Forge 配置 |
| 新建 | `scripts/build-electron.ts` - 构建脚本 |
| 修改 | `package.json` - 添加依赖和脚本 |
| 修改 | `next.config.ts` - 条件性禁用 Vercel 监控 |
| 修改 | `.gitignore` - 添加 Electron 忽略规则 |

## 注意事项

1. **pdfjs-dist Worker**：Electron 中 PDF.js 的 web worker 路径可能需要调整，但通常 standalone 模式下不会有问题
2. **Sentry**：Edge Runtime 配置在 Electron 中不会被加载（正常行为），客户端 Sentry 继续工作
3. **内存状态**：LangGraph 的 `MemorySaver` 在 Electron 中仍然是内存存储，关闭应用后丢失（与当前行为一致）
4. **开发体验**：开发时直接用 `electron:dev` 脚本，会自动启动 Next.js dev server 并打开 Electron 窗口
